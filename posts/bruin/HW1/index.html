Data Manipulation and Visualizing: Temperature at Stations Distinguished by Location and Season

Part 1: Create a Database

I begin by importing the necessary libaries and packages to create a database that consists of three tables: temperatures, stations, and countries. The data for these tables are extracted from csv files. The cleaning process involves some of the column names being renamed and column values adjusted accordingly. sqlite3.connect() points to the database location.

import sqlite3
import pandas as pd
import numpy as np
from matplotlib import pyplot as plt
import plotly.io as pio
pio.renderers.default = "iframe"
from plotly.io import write_html
def prepare_df(df):
    """
    Cleans the dataframe
    """
    df = df.set_index(keys=["ID", "Year"])
    df = df.stack()
    df = df.reset_index()
    df = df.rename(columns = {"level_2"  : "Month" , 0 : "Temp"})
    df["Month"] = df["Month"].str[5:].astype(int)
    df["Temp"]  = df["Temp"] / 100
    
    return(df)

# creating a database in the current directory called temps.db
with sqlite3.connect("temps.db") as conn: 
    df_iter = pd.read_csv("temps.csv", chunksize = 100000)
    for i, df in enumerate(df_iter):
        df = prepare_df(df)
        df.to_sql("temperatures", conn, if_exists = "replace" if i == 0 else "append", index = False)
    stations_csv = "station-metadata.csv"
    stations = pd.read_csv(stations_csv)
    stations.to_sql("stations", conn, if_exists = "replace", index=False)
    countries_url = "https://raw.githubusercontent.com/mysociety/gaze/master/data/fips-10-4-to-iso-country-codes.csv"
    countries = pd.read_csv(countries_url)
    countries.to_sql("countries", conn, if_exists = "replace", index = False)
    
# the with statement closes the connection when the block is exited (so conn.close() statement is not needed)

df.to_sql() writes to a specified table, either temperatures, stations, or countries in temps.db. Becuase the tables for the metadata in our database are from pretty small data sets, there is no need to read it in by chunks. if_exists = “replace” ensures that a new piece is added to the table each time rather than being overwritten.

Part 2: Write a Query Function

Now, I will perform some basic queries on the data tables by joining them based on relational information using INNER JOIN (only rows that have common characteristics similar to the intersection of two sets).

For example, the id of the stations table corresponds to the id of the temperatures table.

The WHERE parameters take the arguments passed in to correspond to the columns of the dataframe.

Instead of using the cursor for prototyping SQL queries, here I use pd.read_sql_query(cmd, conn) as an easier approach.

from climate_database import query_climate_database

def query_climate_database(db_file, country, year_begin, year_end, month):
    """
    Returns a dataframe of the temperatures at stations specified
    by latitude, longitude, and country within a range of months and 
    years.    
    """
    # creating a database in the current directory 
    # inner join (default for pd.merge): shared columns between the two sets
    with sqlite3.connect(db_file) as conn:
        cmd = \
        f"""
        SELECT S.NAME, S.LATITUDE, S.LONGITUDE, C.Name, T.Year, T.Month, T.Temp
        FROM temperatures T
        INNER JOIN stations S ON T.id = S.id
        INNER JOIN countries C on SUBSTR(T.id, 1, 2) = C."FIPS 10-4"
        WHERE T.year >= {year_begin}
        AND T.year <= {year_end}
        AND T.month = {month}
        AND C.name = "{country}"
        """
        df = pd.read_sql_query(cmd, conn)
        df = df.rename(columns={"Name" : "Country"})
        
        return df
        # the with statement closes the connection when the block is exited (so conn.close() statement is not needed)

Part 3: Geographic Scatter Function

I will now address the question:

How does the average yearly change in temperature vary within a given country?

from plotly import express as px
from sklearn.linear_model import LinearRegression

coef() computes an estimate of the year-over-year average change in temperature in each month at each station using linear regression. The coefficient of Year is an estimate of the yearly change in Temp when regressing Temp against Year.

def coef(data_group):
    """
    Returns the first coeffecient of the linear regression model.
    """
    x = data_group[["Year"]] # 2 brackets because X should be a df
    y = data_group["Temp"]   # 1 bracket because y should be a series
    LR = LinearRegression()
    LR.fit(x, y)
    return LR.coef_[0]

min_obs refers to the minimum number of years of data for any given station. Because I am only interested in data of at least min_obs years worth of data in the specified month, all other data should be filtered out using df.transform() plus filtering.

Additional keyword arguments may also be passed to the scatter_mapbox() to control the colormap and mapbox style.

def temperature_coefficient_plot(db_file, country, year_begin, year_end, month, min_obs, **kwargs):
    """
    Returns a figure of the estimated yearly increase in temperature
    of the latitude against longitude in a specified country. 
    """
    query_climate_df = query_climate_database(db_file, country, year_begin, year_end, month) 
    
    query_data_obs = \
    query_climate_df[(query_climate_df.groupby("NAME")["Year"].transform(len)) >= min_obs]
    
    # first coefficient of a linear regression model at that station is computed using the .apply() method
    coefs = query_climate_df.groupby(["NAME"]).apply(coef)
    coefs = coefs.reset_index()
    
    merge_data = pd.merge(coefs, query_climate_df, on = "NAME")
    merge_data["Estimated Yearly Increase (°C)"] = \
    round(merge_data[0], 3)
    
    months = {1:"January", 2:"February", 3:"March", 4:"April", 5:"May",
             6:"June", 7:"July", 8:"August", 9:"September", 10:"October",
             11:"November", 12:"December"}
    
    # first argument must be dataframe
    # lat and lon arguments tell px which columns contain the latitude and longitude coordinates
    # hover_name : what appears when hovering over the plotted points
    # height : aspect ratio
    fig = px.scatter_mapbox(merge_data,
                           lat = "LATITUDE",
                           lon = "LONGITUDE",
                           hover_name = "NAME",
                           color = "Estimated Yearly Increase (°C)",
                           height = 300,
                           opacity = .2,
                            # colorbar is centered at 0 so that the "middle" of the
                            # of the colorbar corresponds to a coefficient of 0
                           color_continuous_midpoint = 0,
                           labels = {"LATITUDE", "LONGITUDE"},
                           title = \
                           f"""
                           Estimates of yearly increase in temperature in {months[month]} for stations in {country}, years {year_begin} - {year_end}
                           """,
                           **kwargs)
    # controls which map tiles are used in the visualization and the amount of whitespace
    fig.update_layout(margin = {"r":0,"l":0,"b":0,"t":50}, font_size = 10)
    return fig

Part 4: Creating Two More SQL Query Functions & Corresponding Figures

Question 1: What are the average temperatures at stations within 0-90 degrees latitude and 0-180 degrees longitude (regions of the Northern Hemisphere) during the months of a given year?

SQL Query Function: Northern Hemisphere Temperatures

This additional SQL query function returns a dataframe of the average temperatures at stations within 0-90 degrees latitude and 0-180 degrees longitude, which represents the regions of the Northern Hemisphere (between the equator and the North Pole), during the months of the specified year of 2020.

def query_northHemi_database(year):
    """
    Returns a dataframe of the average temperatures at stations within
    0-90 degrees latitude and 0-180 degrees longitude, which represents
    the regions of the Northern Hemisphere, during a specified year.
    """
    cmd =\
    f"""
    SELECT S.NAME, S.LATITUDE, S.LONGITUDE, C.Name, T.Year, T.Month, ROUND(AVG(T.Temp), 1) "Average Temperature"
    FROM temperatures T
    INNER JOIN stations S ON T.id = S.id
    INNER JOIN countries C on SUBSTR(T.id, 1, 2) = C."FIPS 10-4"
    WHERE (S.LATITUDE > 0 AND S.LATITUDE < 90) 
    AND (S.LONGITUDE >= 0 AND S.LONGITUDE <= 180)
    AND T.Year = {year}
    GROUP BY S.NAME, C.Name
    """

    df = pd.read_sql_query(cmd, conn)
    df = df.rename(columns={"Name" : "Country"})
    # sort the months by order
    df = df.sort_values(by = "Month")

    return df

Question 2: What are the average temperatures at each station in each country of the Northern Hemisphere during the summer and winter seasons during a given year?

SQL Query Function: Summer and Winter in Northern Hemisphere

Next, building on the previous question as the Northern Hemisphere is still in focus, now I will add an additional component: seasons. Specifically, I am interested in the summer and winter seasons, and the average temperatures associated with each station in each country in the Northern Hemisphere during the summer and winter.

The months of summer are June, July, and August, which can be numerically expressed as 6, 7, and 8, respectively. The months of winter are December, January, and February, which can be numerically expressed as 12, 1, and 2, respectively.

summerWinterMonths = {"Summer": [6,7,8], "Winter":[12, 1, 2]}

A new SQL query function is created to reorganize the data according to the summer and winter months of a specific country and year. Only the summer and winter months are of focus for this query function.

def query_summerWinter_database(season, year):
    """
    Returns a dataframe of the average temperatures at stations of countries 
    still within the Northern Hemisphere during the months of a specified 
    season and year.
    """
    # grab each month by the values associated with the key as inputted by the parameter
    cmd = \
    f"""
    SELECT S.NAME, S.LATITUDE, S.LONGITUDE, C.Name, T.Year, T.Month, ROUND(AVG(T.Temp), 1) "Average Temperature"
    FROM temperatures T
    INNER JOIN stations S ON T.id = S.id
    INNER JOIN countries C on SUBSTR(T.id, 1, 2) = C."FIPS 10-4"
    WHERE T.year = {year}
    AND (T.Month = {summerWinterMonths[season][0]} 
    OR T.Month = {summerWinterMonths[season][1]}
    OR T.Month = {summerWinterMonths[season][2]})
    AND (S.LATITUDE > 0 AND S.LATITUDE < 90) 
    AND (S.LONGITUDE >= 0 AND S.LONGITUDE <= 180)
    GROUP BY S.NAME, C.Name
    """
    
    df = pd.read_sql_query(cmd, conn)
    df = df.rename(columns={"Name" : "Country"})
    df["Season"] = season
#     df = df.sort_values(by = "Month")
#     df["Season"] = list(summerWinterMonths.keys())[list(summerWinterMonths.values()).index(df.get("Month"))]
#     df["Season"] = df["Season"].sort_values(by = "Month")

    return df

Bar Graphs for Both Summer and Winter in Northern Hemisphere

Using the SQL query function created above, a bar graph is created representing the average temperatures of the stations in each country of the Northern Hemisphere during the summer and winter seasons of 2012.

import plotly.io as pio
pio.renderers.default="iframe"

    """
    Returns a figure containing a bar graph of countries against average temperatures at each station

    query_summerWinter = query_summerWinter_database(season, year)

                            color = "Average Temperature",

                            hover_data = ["Country","NAME", "LATITUDE", "Month"],
                            )
    fig.update_layout(margin={"r":0,"t":50,"l":0,"b":0})
    return fig

Potential Issue with Figures:

Unfortunately, one issue with the original color map gradient (color_map = px.colors.diverging.RdGy_r) is that most of the values were either very low which were hard to see if they were associated with a very light color such as yellow. This causes the majority stations with positive temperatures to be difficult to see as positive temperatures are associated with lighter color in the scale. Therefore, I changed the the color_continuous_scale to be “Bluered_r” to be more apparent.

Conclusion:

The query functions and their corresponding figures illustrate how datasets may be manipulated in terms of the variables they depict, the different ways of integrating their tables, and the type of plot used for display. Another takeaway from the resulting visualizations is that collecting larger amounts of data that involve each variable of interest is important as this can provide a better overall picture of the trends inherent in the dataset. With less points to plot, detecting these patterns become much more difficult and therefore breeds a more vague answer to research questions.
